# When To Use What

Use this guide when two skill families both seem plausible. The router remains the safest default:

```text
Use $sj-core-catalog to choose the right SJ skill sequence for this task:
<describe the artifact or decision>
```

## Decision Table

| If the main problem is... | Start with | Add when needed |
|---|---|---|
| Too many features, unclear product job, or bloated roadmap | `$sj-product-simplify-to-one` | `$sj-product-hack-away-unessential`, `$sj-story-sell-the-improvement` |
| The interface needs explanation before users understand it | `$sj-product-speaks-for-itself` | `$sj-ive-humanize-technology`, `$sj-product-telegraph-to-telephone` |
| The product feels cold, technical, or intimidating | `$sj-ive-humanize-technology` | `$sj-product-liberal-arts-technology`, `$sj-story-old-story-new-tool` |
| The physical, sensory, or tactile quality matters | `$sj-ive-material-honesty` | `$sj-ive-care-is-felt`, `$sj-product-beautiful-object-standard` |
| The first user encounter is weak | `$sj-ive-first-touch-moment` | `$sj-product-apple-experience-audit`, `$sj-product-reputation-credits` |
| A redesign may lose what people love | `$sj-ive-future-without-betrayal` | `$sj-people-founder-soul-guardian`, `$sj-strategy-build-for-yourself` |
| A launch sounds like a feature list | `$sj-story-sell-the-improvement` | `$sj-story-one-message-marketing`, `$sj-story-three-act-launch` |
| The demo is brittle or unclear | `$sj-story-demo-rehearsal-loop` | `$sj-product-concrete-artifact-review` |
| Hiring bar or team shape is fuzzy | `$sj-people-a-player-bar` | `$sj-people-small-a-team`, `$sj-people-missionaries-not-mercenaries` |

## Product vs Ive

Use `sj-product-*` when the question is what the product should do, how it should be simplified, or whether the artifact works.

Use `sj-ive-*` when the question is how the product feels, how users first meet it, whether care is visible, whether material or implementation choices support the idea, or how to modernize without losing soul.

## Story vs Product

Use `sj-product-*` before story when the thing itself is still unclear.

Use `sj-story-*` when the thing is clear enough to explain, launch, demo, or sell.

## Strategy vs People

Use `sj-strategy-*` when the portfolio, market, distribution, company design, or failure pattern is the constraint.

Use `sj-people-*` when the decision depends on talent, ownership, trust, feedback, or leadership behavior.

## Learning vs Compounding

Use `sj-learning-*` for personal operating lenses: reading, practice, mortality, endurance, balance, and ego.

Use `$sj-core-compound-learning` when a session created a durable lesson worth saving into a project.

Use `$sj-core-learning-refresh` when saved lessons may be stale, duplicate, weak, or contradictory.

## Anti-Pattern vs Failure Review

Use `sj-anti-*` when you suspect a specific failure mode right now: revenge motive, fake traction, or too much money.

Use `$sj-strategy-failure-apprenticeship` after something has already failed and needs a full autopsy.
