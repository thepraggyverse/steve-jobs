# Care Is Felt

## Core Move

Look for the tiny decisions where users infer whether the maker cared, even when they cannot name the detail consciously.

## Workflow

1. Inspect the artifact for small signals of care or carelessness.
2. Separate functional correctness from felt quality.
3. Prioritize details that users encounter repeatedly or emotionally.
4. Suggest the smallest changes that make care visible or felt.
5. Define how to re-inspect the artifact after the changes.

## Output

Return a compact result with:

- Care signals
- Careless signals
- High-leverage details
- Repair plan
- Reinspection checklist

## Guardrails

- Do not polish meaningless details while core utility is broken.
- Care should serve the user, not the maker ego.
- Name concrete details instead of saying it needs more love.
- Do not use long quotations from the source material.

Use $sj-ive-care-is-felt to find where this product feels cared for or careless.
