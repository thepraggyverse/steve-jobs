# Ask for help

## Core Move

Ask serious builders directly and specifically.

## Workflow

1. Name the decision or obstacle and the specific expertise the other person has.
2. Summarize relevant context, constraints, attempts, and evidence in a form they can scan quickly.
3. Ask one bounded question that permits candid disagreement rather than a vague request to pick their brain.
4. Make the time, format, confidentiality, and desired response explicit while giving an easy decline.
5. Close the loop afterward with the decision taken, result, and appropriate thanks or reciprocity.

## Output

Return a compact result with:

- Specific need
- Compact context
- Bounded ask
- Low-friction request
- Close-the-loop plan

## Guardrails

- Do not disguise a large unpaid project as quick advice.
- Avoid asking for help before making a reasonable first attempt.
- Respect a no or silence without pressure.
