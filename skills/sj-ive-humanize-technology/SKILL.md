# Humanize Technology

## Core Move

Translate technical capability into an object, interface, or moment that feels understandable, calm, and human.

## Workflow

1. Identify what currently feels abstract, technical, or intimidating.
2. Name the human confidence the product should give the user.
3. Replace machine-centered language, structure, or interaction with human-centered cues.
4. Find one sensory or interaction detail that can make the technology feel warmer.
5. Return a before-and-after improvement plan.

## Read References

- Read `references/sj-ive-design-studio.md` when this skill needs source grounding or deeper examples.
- Read `references/sj-source-map.md` when this skill needs source grounding or deeper examples.

## Output

Return a compact result with:

- Cold or abstract points
- Desired human feeling
- Language changes
- Interaction changes
- Next prototype

## Guardrails

- Do not hide important power or risk behind friendliness.
- Do not add fake warmth that slows expert users.
- Favor clarity and care over cuteness.
- Do not use long quotations from the source material.

## Example Prompt

Use $sj-ive-humanize-technology to make this interface feel less cold and more approachable.
