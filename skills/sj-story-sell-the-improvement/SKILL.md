# Sell the improvement

## Core Move

Sell the better life, not the product feature.

## Workflow

1. Translate each feature into the user behavior, friction, or outcome it changes.
2. Choose the improvement with the clearest before-and-after contrast for the target user.
3. Gather proof from a real task, measurement, artifact, or customer observation.
4. Write the old experience, the improved experience, and the mechanism connecting them.
5. Remove features that do not strengthen the transformation and end with a concrete invitation.

## Output

Return a compact result with:

- Feature-to-outcome map
- Chosen improvement
- Before and after
- Proof
- Transformation pitch

## Guardrails

- Do not promise an outcome the feature cannot cause.
- Avoid generic benefits such as faster or easier without a task and baseline.
- Keep necessary technical detail when it establishes credibility.
