# Prototype Volume

## Core Move

Move from opinion to comparison by creating enough concrete variants for taste, function, and tradeoffs to become visible.

## Workflow

1. Name the decision being argued in abstract terms.
2. Define the smallest prototype that can make the decision visible.
3. Generate a set of contrasting variants, not one precious version.
4. Create selection criteria tied to the product story and user experience.
5. Pick, combine, or reject variants based on what the artifacts reveal.

## Output

Return a compact result with:

- Abstract debate
- Prototype unit
- Variant set
- Selection criteria
- Decision from artifacts

## Guardrails

- Do not confuse prototype volume with unfocused thrashing.
- Each prototype should test a distinct bet.
- Choose after inspection, not after defending personal taste.
- Do not use long quotations from the source material.
